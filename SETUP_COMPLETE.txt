## Vulnerable Code Highlighted

### SQL Injection in Login (login.php, lines 16-18)
```php
$username = $_REQUEST['username'];
$password = $_REQUEST['password'];
$sql = "SELECT * FROM " . $htbconf['db/users'] . " where " . 
       $htbconf['db/users.username'] . "='$username' and " . 
       $htbconf['db/users.password'] . "='$password'";
```
**Issue**: Username and password directly interpolated without escaping
**Attack**: `username=' OR '1'='1` → Bypasses authentication

### Stored XSS in Transfers (htbdetails.page, line 45)
```php
$transfersStr .= "<td>".$row[5]."</td>\n";  // row[5] is remark
print $transfersStr;
```
**Issue**: Remark field echoed without HTML escaping
**Attack**: `<img src=x onerror="alert('XSS')">` → Executes in browser

### RCE via Regex /e Modifier (htbdetails.page, lines 56-60)
```php
$replaceWith = "preg_replace('#\b". str_replace('\\', '\\\\', $http['query']) 
               ."\b#i', '<span class=\"queryHighlight\">\\\\0</span>','\\0')";
$transfersStr = preg_replace('#(\>((?>(([^><]+|(?R))))*\<))#se',$replaceWith,'>'.$transfersStr.'<');
```
**Issue**: User input in `/e` modifier replacement string (executes as PHP)
**Attack**: `query=" . system('whoami') . "` → Runs system commands

### Parameter Tampering (htbloanconf.page, line 24)
```php
$sql="update ".$htbconf['db/accounts']." set ".$htbconf['db/accounts.curbal'].
     "=".$htbconf['db/accounts.curbal']." + \"".($http['loan'])."\", ...";
```
**Issue**: Loan amount directly concatenated from user input
**Attack**: Intercept request, change `loan=1000` to `loan=999999`
